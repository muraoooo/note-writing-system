# noteアルゴリズム — 何が露出を決めるかを理解する

## 基本構造(2026年時点の観察ベース)

noteの露出は大きく以下から発生する:

| 流入経路 | 寄与度 | 攻略法 |
|---|---|---|
| Google検索 | ★★★★ | SEO対策・キーワード設計 |
| note内検索 | ★★★ | タイトル・タグ最適化 |
| ハッシュタグページ | ★★★ | 公式タグ・適切なタグ選定 |
| おすすめ・タイムライン | ★★ | 公開直後の初速エンゲージメント |
| SNS外部流入 | ★★ | Twitter/X連携・シェア導線 |
| マガジン購読 | ★ | フォロワー基盤 |

## 評価される指標(推定)

1. **公開後3時間のエンゲージメント率**(スキ + コメント + シェア / ビュー)
2. **読了率 / 滞在時間**
3. **保存数(マイマガジン追加)**
4. **コメント数**
5. **記事の更新頻度(週1以上が目安)**
6. **フォロー転換率**

## 公開直後の「初速」が最重要

noteは**公開から3時間が勝負**。ここで反応がないと、おすすめに乗らず埋もれる。

### 初速を作る打ち手
- 公開時刻を**平日 朝7-8時 or 夜21-23時**に設定(可処分時間の山)
- 公開と同時にTwitter/Xで告知
- 仲間内にDMで告知(同業者コミュニティを持つ)
- 自分のマガジンに即時追加

## 滞在時間を伸ばす施策

- 1記事の文字数: **2,500〜4,500文字**(短すぎても長すぎてもNG)
- 画像・コード・表を**3個以上**挿入
- 目次が機能する見出し設計(7〜10個)
- 冒頭3行で離脱させない(`記事設計/リード文.md`参照)

## 読了率を上げる

- スマホでの可読性(段落3行以内)
- 中盤に「ここからが本題」的な再フック
- 最後まで読む価値の予告(「最後にハマりポイントもまとめます」)

## アルゴリズムが嫌う行動

- ❌ 短期間に大量投稿(スパム判定の可能性)
- ❌ タグの大量・無関係付与
- ❌ 同じタイトル形式の連続(機械生成と判定)
- ❌ 外部リンクだらけの記事(離脱誘発)
- ❌ コピペ・転載感の強い文章

## フォロワー獲得の方程式

```
フォロワー = (1記事の到達ビュー × フォロー転換率)の累積
```

- 到達ビューを上げる: タイトル・タグ・公開時刻
- フォロー転換率を上げる: ポジショニング明確化(ファイル参照)
- **目安**: 100ビューあたりフォロー1〜2人なら健全

## 月次でモニタすべき数値

| 指標 | 健全ライン |
|---|---|
| 1記事平均ビュー | 500以上 |
| スキ率 | 2%以上 |
| コメント率 | 0.5%以上 |
| フォロワー月間増加 | 10人以上 |
| 全体ビュー月間 | 5,000以上 |

## 現状とのギャップ(2026年5月1日時点)

| 指標 | 現状 | 健全ライン | ギャップ |
|---|---|---|---|
| 1記事平均ビュー | 32 | 500 | -468 |
| スキ率 | 0.5% | 2% | -1.5pt |
| コメント率 | 0% | 0.5% | -0.5pt |

→ **タイトル + リード + ハッシュタグ + 公開時刻** の4点同時改善で2-3か月で伸ばす。
