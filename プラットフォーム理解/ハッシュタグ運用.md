# ハッシュタグ運用 — note内流入の主導線

## なぜハッシュタグが重要か

note には**ハッシュタグごとのページ**があり、そこで読者が記事を発見する。
検索流入と並ぶ**主要な流入経路**であり、適切なタグ設計だけでビューが2〜3倍になることもある。

## タグ数の最適解

- **5〜7個**(noteの設定上限は10個だが、絞った方が効果的)
- 多すぎるとスパム判定 / 関連性が薄まる

## タグ選定の3階層構成

### 第1階層: 公式タグ・大規模タグ(1〜2個)
- 投稿数が10万件以上のメジャータグ
- 例: `#AI` `#プログラミング` `#エンジニア`
- メリット: 流入の母数が大きい
- デメリット: 競合が多く埋もれやすい

### 第2階層: 中規模タグ(2〜3個)
- 投稿数が1,000〜10万件
- 例: `#ローカルLLM` `#ChatGPT` `#OSS`
- ここが**最も流入効率が良い**ゾーン

### 第3階層: ニッチタグ(1〜2個)
- 投稿数が100〜1,000件
- 例: `#akira-papa` `#Llama3` `#MITライセンス`
- 競合が少なく上位表示されやすい

## 既存記事の推奨タグ例

### ローカルLLM akira-papa 記事
```
#AI #ローカルLLM #ChatGPT代替 #Mac #akira-papa
```
- AI(大) / ローカルLLM(中) / Mac(中) / ニッチ2つ

### MITライセンス商用記事
```
#プログラミング #OSS #ライセンス #商用利用 #MITライセンス
```

### MP3 ChatGPT記事
```
#ChatGPT #AI #文字起こし #業務効率化 #音声AI
```

## 公式お題(noteコンテスト)タグの活用

note公式が運営する「お題」タグに参加すると:
- 公式ページに掲載されるチャンス
- 通常より露出が増える
- 月1〜2本は公式お題に乗せる運用がおすすめ

確認方法: note公式の「コンテスト・お題」ページを定期チェック。

## やってはいけないタグ運用

- ❌ 関係ないバズワードを付ける(信頼低下)
- ❌ 全記事で同じタグを使い回し(タグページで自分同士が競合)
- ❌ 10個フルに付ける(広すぎて埋もれる)
- ❌ 空白入りタグ(noteで認識されない)
- ❌ 英語/日本語表記の不統一(`#AI` と `#人工知能` を別記事で散発的に使う)

## タグの統一ルール

連載感を出すため、**シリーズ共通タグ**を必ず1つ含める:

| シリーズ | 共通タグ |
|---|---|
| ローカルLLM検証 | `#ローカルLLM検証ブログ` |
| OSS削除手順 | `#OSSアンインストール` |
| ライセンス解説 | `#OSSライセンス入門` |

これでタグページから自分の他記事への回遊が生まれる。

## タグ運用の月次レビュー

- どのタグから何ビュー来たか(noteのダッシュボードで確認)
- 流入が0だったタグは差し替え候補
- 新興タグ(突然投稿が増えてるタグ)は早期参入で上位表示しやすい
