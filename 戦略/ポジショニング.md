# ポジショニング — 「あなたから読む理由」を作る

## 結論

note市場は飽和している。**「AIの使い方を発信する人」は1万人いる。** その中で読まれるには、テーマ × 切り口 × 人格 の交差点で唯一になるしかない。

## ポジショニング設計の3軸

```
        専門性(深い)
          │
          │
浅く広く ─┼─ 深く狭く
          │
          │
        実践性(動くコードがある)
```

### 推奨ポジション(本リポジトリ筆者向け)

**「自分のPCで動かす派の検証ブロガー」**

- 一般受けする「ChatGPTで業務効率化」系には参入しない(レッドオーシャン)
- ローカルLLM・OSS・ライセンス・削除手順 など「実機検証が必要な領域」に絞る
- 競合が「やってみた」で終わるのに対し、**「失敗例」「削除手順」「ライセンス注意点」までセットで書く**
- これは現在の記事ラインナップと一致しているので、ブレずに深堀りすべし

## ポジショニング・ステートメント(更新可能)

> **「クラウドAIに月額払いたくない人のための、ローカルAI実機検証ブログ」**
>
> ローカルLLM・OSSツールを実際にPCに入れて動かし、入れ方だけでなく、消し方・ライセンス・落とし穴までセットで書きます。

このステートメントを各記事の冒頭またはプロフィール欄に入れる。読者の頭に「この人は◯◯の人」とラベルが貼られる状態を作る。

## 差別化の4要素(他人より1つでも勝てばよい)

| 要素 | 自分の強み? | 補強方法 |
|---|---|---|
| 速さ | 新ツールが出てから48時間以内 | RSS/Trending監視 |
| 深さ | 公式ドキュメント+ソースコードまで読む | 引用と参照リンク |
| 失敗体験 | 自分が踏んだ罠を晒す | 「やらかしたこと」セクション必須化 |
| 削除/撤退手順 | 「入れた後どうする」を書く人は少数 | 全ツール記事に「アンインストール」章を |

## やらないことを決める

- ❌ ChatGPT GPTs作ってみた系(競合多すぎ)
- ❌ プロンプトエンジニアリング一般論(陳腐化が早い)
- ❌ ノーコード入門(キーワード被害が大きい)
- ✅ ローカル動作・OSS・実機検証・後始末

## ブランド化のための運用ルール

1. **アイコン・ヘッダー画像は1年変えない**(認知の固定化)
2. **記事末尾の署名フォーマットを固定**(「〇〇でした。次回は△△を試します」)
3. **シリーズ化**(akira-papa編、OpenClaw編 など連番で並べる)
4. **プロフィールに「やってること」を箇条書き3つ**(自己紹介より検索結果)
