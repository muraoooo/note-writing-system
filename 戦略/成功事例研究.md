# 成功事例研究 — 「伸びる記事」の共通パターン

## note上で伸びる記事の典型形(2026年現在)

### パターン1: 「やってみた + 数値結果」型
- タイトル例:「ChatGPT Plusを3か月使った結果、月◯時間削減できた件」
- 構造:背景 → 検証方法 → 数値 → 学び
- 効果:具体性で信頼を獲得、ブクマされやすい

### パターン2: 「失敗 → 解決」型
- タイトル例:「初めてローカルLLMを動かしたら3回詰まった話」
- 構造:期待 → 失敗 → 試行錯誤 → 解決
- 効果:同じ失敗を恐れる読者の保険記事になる

### パターン3: 「網羅まとめ」型
- タイトル例:「ローカルLLMツール12選を実際に試した比較表」
- 構造:目的 → 比較表 → 各ツール詳細 → 推奨
- 効果:検索流入 + ブクマ + 引用

### パターン4: 「内幕暴露 / 業界裏話」型
- タイトル例:「OSSライセンスで会社が訴えられた事例から学ぶ」
- 構造:事件 → 何が原因だったか → 教訓
- 効果:感情を動かしシェアされる

### パターン5: 「テンプレ・コピペ」型
- タイトル例:「コピペで動く Llama 3 ローカル起動コマンド集」
- 構造:前提 → コマンド一覧 → 注意点
- 効果:保存される=長期的にビュー

## 共通する成功要素

伸びている記事を50本観察すると、以下が99%含まれる:

1. **タイトルに数字** (「3つの」「12選」「30分で」)
2. **冒頭3行で読者の悩みを言語化**
3. **見出しが7〜10個**(目次が機能する)
4. **画像/コードブロックが3個以上**
5. **締めに次のアクションを提示**(「他の記事も読む」「フォロー」「コメント」)

## 「自分が書ける成功パターン」の選択

5パターンすべてを書ける必要はない。**自分の強みと相性のいい2パターンを軸にする。**

推奨:
- **メイン**: パターン2(失敗→解決)— 検証ブロガーのポジションと一致
- **サブ**: パターン5(テンプレ・コピペ)— 保存数を稼ぎ長期ビュー獲得
