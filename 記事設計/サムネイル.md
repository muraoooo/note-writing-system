# サムネイル — タイムラインで指を止めさせる視覚装置

## なぜ重要か

noteのタイムライン・おすすめ・検索結果——どこでも記事は**画像 + タイトル**のセットで表示される。
タイトルだけ磨いてサムネイルがデフォルトのままだと、目立つ競合に埋もれる。**スキ率を倍にする最も簡単な施策がサムネイル設計。**

## サムネイル設計の3原則

### 1. 1.91:1 の比率を死守
- noteの推奨サイズ: **1280 × 670 px** 前後
- これ以外だと自動トリミングで文字が切れる

### 2. タイトルの一部を画像内テキストに
- タイムライン上で**タイトル+画像内文字でダブル訴求**
- 例:画像中央に大文字で「30分で動く」+ タイトルに技術名

### 3. 色は「2色 + アクセント1色」
- 背景:濃い色(黒・ネイビー・ダークグリーン)
- 文字:白または黄色
- アクセント:オレンジか赤(矢印・下線・ハイライト用)

## ジャンル別の定番デザイン

| ジャンル | 構図 | 例 |
|---|---|---|
| ハウツー | 左テキスト・右イラスト | 「30分で動かす」+ ロボのイラスト |
| 比較記事 | 左右2分割 | A vs B |
| 失敗談 | 「やらかした」を大きく | 大文字で警告風 |
| まとめ系 | 数字を巨大に | 「12選」を画面1/3で表示 |

## 制作ツール推奨(無料)

1. **Canva** — テンプレ豊富・note向けサイズあり・最速
2. **Figma** — 自分のテンプレを再利用できる(連載向け)
3. **Excalidraw** — 手書き風・技術記事と相性良い
4. **Bing Image Creator / DALL-E** — オリジナル画像を生成

## 連載化のためのテンプレ運用

**全記事で同じレイアウトを使う。** ロゴ位置・フォント・配色を固定すると、タイムラインで「あ、この人の記事だ」と一瞬で認識される。これがフォロワー獲得の原動力。

推奨テンプレ構成:
```
┌────────────────────────────┐
│  [カテゴリラベル]            │  ← 「ローカルLLM #03」など
│                              │
│  メインコピー(大)           │  ← タイトルの核心
│  サブ(小)                   │
│                              │
│              [筆者ロゴ/アイコン]│
└────────────────────────────┘
```

## やってはいけない

- ❌ デフォルトの自動生成画像のまま公開
- ❌ 文字情報ゼロの風景写真
- ❌ AI生成画像で「手の本数がおかしい」状態を放置
- ❌ 文字が小さくスマホで読めない
- ❌ 画像内に長文(15文字以下に抑える)

## チェックリスト

- [ ] 1280×670 で出力したか
- [ ] スマホサイズで文字が読めるか
- [ ] タイトルと画像内文字が重複しすぎず補完関係にあるか
- [ ] 連載テンプレと統一されているか
- [ ] アイキャッチに筆者の識別要素(色・ロゴ)があるか
